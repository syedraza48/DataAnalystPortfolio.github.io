Data Cleansing
Using Microsoft SQL Server

In this project, our goal was to clean the data that is being imported and make it more useable and easy to access. This is a part of the ETL process. We accomplished this by the following- we used the CONVERT function to standardize the format of the date column. We populated the property address column wherever there were null values before breaking the address, city, state into separate columns using SUBSTRING, CHARINDEX, PARSENAME, REPLACE functions. We then changed 'Y' 'N' values to 'Yes' 'No' using CASE statements. We removed duplicates using Row_Number, a CTE (Common Table Expression), and a Window Function/Partition By. Finally, we deleted some useless columns that were not needed. Please click on the button below to view each query and the gradual process in my GitHub respository.

NBA Player & Team FG% Analysis
Using SQL and Tableau

In this project, we joined two tables in PostgreSQL and displayed the results in a basic bar graph using Tableau visualization tool. The data was extracted from ESPN.com using Microsoft Excel. The excel files were then used to load data into our database where we created two tables- "player_stats" and "team_stats." The query was stored in a view for easier access. We then took our tabular results from the query and created a simple Tableau dashboard that shows each top 50 player's FG% next to their team's collective FG% with the league average FG% going accross the chart.
This data is valid up to Jan 23, 2022. We will update the Tableau dashboard every Sunday night to reflect all changes to the top 50 pool. This project will eventually attempt to predict who the MVP front runners are when we start adding more statistcs like W/L% and team rank.

Tableau Dashboard

In this project we are creating a dashboard analyzing the data of Air BnB listings of a city.
Use case of this data- if someone wants to buy a home and use it for Air BnB income, where should they buy it and how much can they charge? We will optimize this answer for maximizing profits.
The graphs with color represent price by zipcode. The first is in bar graph format while the other is in geographical format.
The line graph shows revenue of Air Bnb listings by time of year. More profits can be made during the summer and late year holiday travel time. The beginning of the year shows that listing prices are relatively lower.
The bar graph at the top of the dashboard shows the average price of Air Bnb listings by bedroom. The more bedrooms available the more you can charge for the listing. Something to consider if one is investing in this zipcode.
Finally, the numerical table at the top right shows the count of listing by bedroom.
We are now ready to build on top of this analysis and extract more infomation if necessary.
